# CS 330 Portfolio Reflection

## How do I approach designing software?

When I design software, I first think about what the final project needs to do and what the user should see. For my 3D scene, I started by breaking the real-world object into simple shapes such as cylinders, boxes, cones, and planes. This project helped me improve my design skills because I had to plan the scene before coding it. I followed an iterative design process where I built one part at a time, tested it, and made changes when something did not look right. I can use this same approach in future projects by planning first, building in smaller sections, and improving the design through testing.

## How do I approach developing programs?

When developing programs, I try to work step by step instead of doing everything at once. In this project, I used new strategies such as creating objects from basic 3D shapes, applying transformations, adding textures, and setting up lighting. Iteration was a big part of my development because I had to keep testing the scene, fixing problems, and adjusting the objects until they looked correct. Throughout the milestones, my coding approach improved because I became more comfortable organizing code, using shaders, and understanding how transformations affect objects in a 3D environment.

## How can computer science help me in reaching my goals?

Computer science can help me reach my goals by giving me skills that apply to both school and my future career. Computational graphics taught me how math, programming, and visual design work together to create interactive 3D scenes. These skills can help me in future courses because I now have more experience with problem-solving, debugging, and writing organized code. Professionally, this project gave me a better understanding of how graphics, visualization, and software development can be used in areas such as simulations, game design, cybersecurity tools, and other computer science fields.
